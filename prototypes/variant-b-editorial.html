"What depends on what?"

A question nobody at Meta could answer. Dependency-related failures were ~15% of all top-severity incidents, with no single source of truth. Five systems fixed that — each one making the next possible.

1

See it

The dependency graph in Rust — a streaming pipeline fusing 6 telemetry sources at ~20 GB/s into one canonical map.

0 hidden dependencies surfaced
2

Route it

A notification + protection program so the owning team hears about a risky dependency early.

0 services protected
3

Explain it

An LLM agent harness over 16 purpose-built tools explains any dependency pair — structured tool calls, not embeddings.

0 pairs · ~100% coverage · ~$0.15 each
4

Rank it

A hard/soft classifier separates dependencies that take the caller down from those that degrade gracefully.

5

Prove it

A fault-injection harness disables one dependency at a time and records which SLO degrades. Observed outcomes become ground-truth labels.

eliminated risky, revenue-critical dependencies

The outcomes from prove it feed back into rank it — so the platform corrects itself against reality instead of plateauing.

That last loop is the whole point. It is how the platform kept getting more accurate, and how it chipped into ~15% of company-wide top-severity incidents.
